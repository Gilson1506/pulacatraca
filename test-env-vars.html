🧪 Teste de Variáveis de Ambiente - Vite

📝 Instruções:

  1. Certifique-se de que o frontend está rodando (npm run dev)
  2. Acesse: http://localhost:5173
  3. Abra o Console do Navegador (pressione F12)
  4. Cole e execute o código abaixo no console

🔍 Código para executar no Console:

console.clear();
console.log('%c🔍 TESTE DE VARIÁVEIS DE AMBIENTE', 'font-size: 16px; font-weight: bold; color: #3b82f6');
console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n');

// Testar variáveis de ambiente
const webhookUrl = import.meta.env.VITE_PAGBANK_WEBHOOK_URL;
const supabaseUrl = import.meta.env.VITE_SUPABASE_URL;
const supabaseKey = import.meta.env.VITE_SUPABASE_ANON_KEY;

console.log('1️⃣  Webhook URL do PagBank:');
console.log('   Valor:', webhookUrl);
console.log('   Tipo:', typeof webhookUrl);
console.log('   Válido:', webhookUrl && webhookUrl.startsWith('https://') ? '✅ SIM' : '❌ NÃO');
console.log('');

console.log('2️⃣  Supabase URL:');
console.log('   Valor:', supabaseUrl);
console.log('   Válido:', supabaseUrl && supabaseUrl.startsWith('https://') ? '✅ SIM' : '❌ NÃO');
console.log('');

console.log('3️⃣  Supabase Anon Key:');
console.log('   Valor:', supabaseKey ? supabaseKey.substring(0, 30) + '...' : 'undefined');
console.log('   Válido:', supabaseKey && supabaseKey.length > 20 ? '✅ SIM' : '❌ NÃO');
console.log('');

console.log('━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n');

// Resultado final
if (webhookUrl && webhookUrl.startsWith('https://') && webhookUrl.includes('ngrok')) {
    console.log('%c✅ WEBHOOK URL CONFIGURADA CORRETAMENTE!', 'color: #10b981; font-weight: bold; font-size: 14px');
    console.log('%c   O PagBank vai receber: ' + webhookUrl, 'color: #059669; font-size: 12px');
} else if (webhookUrl === 'http://localhost:3000/api/payments/webhook') {
    console.log('%c❌ PROBLEMA: Ainda está usando LOCALHOST!', 'color: #ef4444; font-weight: bold; font-size: 14px');
    console.log('%c   O frontend NÃO foi reiniciado corretamente.', 'color: #dc2626; font-size: 12px');
    console.log('%c   SOLUÇÃO: Feche o navegador, limpe o cache, reinicie o frontend.', 'color: #dc2626; font-size: 12px');
} else {
    console.log('%c❌ VARIÁVEL NÃO DEFINIDA!', 'color: #ef4444; font-weight: bold; font-size: 14px');
    console.log('%c   Verifique se o arquivo .env existe na raiz do projeto.', 'color: #dc2626; font-size: 12px');
    console.log('%c   Verifique se o frontend foi reiniciado após criar o .env.', 'color: #dc2626; font-size: 12px');
}

console.log('\n━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━');

⚠️ SE A VARIÁVEL AINDA ESTIVER COM LOCALHOST:

  1. Feche TODOS os navegadores (Chrome, Edge, etc.)
  2. No terminal do frontend, pressione Ctrl+C
  3. Limpe o cache: Remove-Item -Path "node_modules\.vite" -Recurse -Force
  4. Inicie novamente: npm run dev
  5. Abra em modo anônimo/privado
  6. Teste novamente

✅ SE A VARIÁVEL ESTIVER CORRETA (ngrok):

Tente criar um pedido PIX e o erro "invalid notification url" deve desaparecer!
